# Usability Evaluation

## Evaluation Method
The MindBridge Figma prototype was evaluated using Nielsen’s Ten Usability  Heuristics. Each team member reviewed the prototype independently and  identified usability issues across the main user flows, including appointment  booking, mood tracking, login/register and resource browsing.  The findings were consolidated and rated using a four-point severity scale:
- 1 = Cosmetic issue
- 2 = Minor usability issue
- 3 = Major usability issue
- 4 = Usability catastrophe

## Priority Fixes Completed
The two major usability issues were prioritised because they affected core user  tasks.

### Mood Tracker Chart
The mood tracker chart was updated with a labelled x-axis showing the days of  the week and a y-axis showing the mood scale from 1 to 5. This made the weekly  trend easier for users to understand without needing extra explanation.

### Booking Form Optional Field
The “Reason for Visit” field was updated with an “Optional” label and a short  reassurance note. This reduced the chance of users abandoning the booking  process because they thought sensitive information was required.

## Issues Planned for Next Sprint
The remaining issues were classified as minor or cosmetic. These will be addressed in the next design sprint:
- Add format labels to resource cards.
- Add visible login error states.
- Add active-state styling to the navigation bar.
